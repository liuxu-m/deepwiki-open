## 当前未解决的核心问题

### 问题：生成后的 Wiki 仍大量空页
当前最关键的问题仍然存在：
- 重新生成后的缓存文件 `~/.adalflow/wikicache/deepwiki_cache_github_AsyncFuncAI_deepwiki-open_zh.json` 中，大部分页仍为：
  - `content = ""`
  - `validation_failed = true`
  - 且没有 `raw_content`
- 浏览器中表现为：
  - 页面树可点
  - 但只有少数页面有正文，其它页实际为空

### 已确认的事实
1. 本地源码里，`api/task_worker.py` 的失败分支已经改成：
   - `content = page_content`
   - `raw_content = page_content`
2. 但最新任务产物 `~/.adalflow/wikicache/tasks/<task_id>/wiki_output.json` 中，失败页仍只有：
   - `content`
   - `validation_failed`
   - `validation_reason`
   没有 `raw_content`
3. 这说明：
   - 当前运行时生成链路与本地代码状态不一致，或
   - 存在另一条旧路径仍在写失败页为空内容

## 最需要继续查的方向

### 优先级 1：确认运行时到底执行了哪条失败页写入逻辑
建议下一个接手的人直接排查：
- `api/task_worker.py` 的失败分支是否真的被当前运行进程执行
- `save_wiki_output()` 调用前的 `generated_pages` 内存内容是否包含 `raw_content`
- `_normalize_wiki_output()` 是否在某种情况下遗漏 `raw_content`
- 是否存在其它旧 worker 进程 / 旧路径仍在写 `wiki_output.json`

### 优先级 2：不要再优先修前端
前端现在已经能正常：
- 切页
- 渲染 markdown
- 显示 warning banner
- 处理后台任务显示

所以当前空页问题不应继续优先怀疑前端，而应集中排查后端任务产物写入链路。
